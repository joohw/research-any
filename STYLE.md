# RssAny 前端视觉风格（理念）

面向 `webui/` 的设计方向说明。**具体色板、字号与变量名以 `webui/src/app.css` 为准**；本文只描述原则与取舍，不罗列 CSS。

---

## 总体方向

- **Linear 系暗色工具型 UI**：低噪声背景、细边框与弱分隔、单一强调色用于交互焦点（链接、主按钮、激活态），避免彩虹式装饰。
- **信息优先**：排版与对比度服务于阅读与扫视；装饰性元素让位于内容与数据。
- **同一套语义 token**：颜色与圆角通过 CSS 变量表达语义（背景、前景、边框、弱化文字、危险/成功等），页面与组件避免散落硬编码色值；品牌 OAuth 图标等例外。

---

## 层次与密度

- **克制**：区块之间用边框与留白区分，而非大块色面；列表与面板保持中等信息密度，避免拥挤也避免过度留白。
- **对齐主栏**：应用壳层为内容区提供统一的最大宽度约束；各页在 `main` 内占满可用宽度，**不在页面内再套一层更窄的「假侧栏」或左右竖线**，避免与 feeds 等主流程页错位。

---

## 字体与文案

- **系统字体栈**：正文与 UI 使用无衬线系统字体，保持加载快、观感中性。
- **层级靠字重与颜色**：标题与正文用字重和 `muted` 层级区分，而非多种字体家族。
- **界面文案默认英文**（面向该版本的 UI 字符串）。

---

## 交互与组件习惯

- **图标**：`lucide-svelte`，尺寸与线宽与顶栏等已有用法保持一致。
- **反馈**：短提示复用项目内既有 toast 等机制，避免每页自造一套。
- **可访问性**：语义结构（标题层级、列表、表单标签）优先于纯 div 堆砌。
